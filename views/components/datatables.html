1. Introduction

A data table contains a header row at the top that lists column names, followed by rows for data.

Checkboxes should accompany each row if the user needs to select or manipulate data.

Data tables may include:

  • Three or more columns of data
  • A corresponding visualization
  • The ability for users to query and manipulate data
2. Intraction
  • Row hover: grey_200
  • Row selected: grey_100
  • Sorted: sorted column should have a darker text than unsorted columns
  • Long column title: There are two options to handle this:
    • Display the full column name and enable horizontal scrolling in the table container
    • Shorten the column name and display it in full on hover (with a tooltip)
3. Action Header

Some tables may require headers with actions instead of titles. In this case you may either:

  • Display persistent actions
  • Display a contextual header that activates when items are selected

For Action buttons:

  • Use borderless buttons
  • Enable upon item selection
4. Specs
