1. Introduction

A menu is a temporary piece of material that appears upon interaction with a button, action, pointer, or other control. It contains at least two menu items. Each menu item consists of a discrete option or action that can affect the app, the view, or selected elements within a view. They display a list of choices, with one choice per line.

2. Usage
2.1 Menu label

The label of a button or control should concisely and accurately reflect the items within the menu. Menu bars typically use single words as labels, like “file,” “format,” and “edit.” Other contexts may require longer labels. When labelling a menu, try to make the label as much descriptive as possible.

2.2 Scrolling

If the height of a menu prevents all menu items from being displayed, the menu can scroll internally.

2.3 Cascading

For menus that cascade, position the menus vertically and horizontally based on their proximity to screen edges.

3. Menu items
3.1 Single-line item

Each menu item is limited to a single line of text that describes the action it will perform when selected.

Menu items may also contain:

  • Icons and tooltip text
  • Controls like checkbox to indicate multiple selected items
3.2 Item ordering

In default, menu items should be sorted alphabetically. However, if there is a good reason, menus with static content can have the most frequently used menu items placed at the top of the menu.

3.3 Item nesting

Menu items can reveal nested submenus. Ideally, limit nesting to one level deep, as it can be difficult to navigate multiple nested submenus.

4. Behavior
4.1 Location

For navigation menu and simple menu, they should be positioned 90% below the menu triggering element.



For menus in a form, they should be positioned over their triggering elements such that the currently selected menu item appears on top of the triggering element.
