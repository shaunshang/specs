Empty State
  • 1. Introduction
  • 2. Usage
1. Introduction

Empty states occur when an item’s content can’t be shown.



A list that doesn’t contain any items, or a search that doesn’t display any results, are examples of empty states. Although these states aren’t typical, they should be designed to prevent user confusion.

2. Usage
2.1 Empty results

Empty state display should:

  • be subtle and neutral with respect to the background
  • convey the purpose and potential of the app in a lively way
  • have a positive tone of message
  • be consistent with the brand
  • convey the purpose of the app without appearing to be actionable
2.2 Start content

Empty state doesn’t only occur when there is no results. It also occurs for starter content. For example, when a new user first comes to a page which is waiting for further actions.


For this, we recommend:

  • use content that has broad appeal and demonstrates primary features
  • if possible, provide content that’s personalized
2.3 Educational content

If the purpose of the screen isn’t easily conveyed through an image and a message, consider showing educational content instead. And we recommend make it:


For this, we recommend:

  • helps users understand what they’ll be able to to do on this screen once it has content
  • possible to dismiss or skip this content
  • brief
2.4 Result matching

There is another situation where may involve the empty state. And that is a search result list.


When nothing exactly matches the user’s query, instead of showing “no result”, are there any result for a query spelled slightly differently? If so, then show the results, as they may help an user find what they’re after.


With this approach, clearly convey in a heading above the results that this content shouldn’t be mistaken for a match to actual query results.
