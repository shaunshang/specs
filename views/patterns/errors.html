1. Introduction

Errors occur when an app fails to complete an expected action.


Some examples of errors include:

  • When user input is not understood
  • An app failing to load
  • Incompatible operations are run concurrently

So to sum up those examples, we can categorize those errors into types:

  • User input errors
  • App errors
  • Incompatible state errors
2. Usage

As errors do occur, sometimes by the users, and sometimes by the app itself. We should try to minimize errors by designing apps that make it easy for users to input information flexibly. Because we can’t forecast or control user from what they do, but we can make the app to be as much flexible as possible. By flexible we mean that:

  • apps should accept common data formats that use affordances to improve user understanding
  • once errors occurred, apps should address them clearly
3. User input errors

Help users fix input errors as soon as they are detected. Disable the submission of a form if errors are detected, and if detected only after form submission, clearly explain the error and how to fix it.

3.1 Text field input

Show error text only after user interaction with a field. If the user inputs incorrect data, helper text may transform into error text. (Minimize form text to the essentials. Not every text field needs helper and/or error text.)

3.2 Color

It’s recommended that you use a contrasting color for error states, such as a warmer hue like red or orange. Error text should be legible, with noticeable contrast against its background color.

3.3 Style
  • Line height: 20px
  • Font size: 11px
  • Padding: 0 4px
  • Display: block
